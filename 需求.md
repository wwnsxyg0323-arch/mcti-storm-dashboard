Build a clean, modern web app for analyzing storm overflow data from two Excel workbooks.

Project goal:
Create an interactive data visualization website for a student engineering project about storm overflow treatment at a sewage treatment works. The website should help explain the flow conditions, overflow patterns, water quality patterns, and simple future/scenario analysis. This is a data analysis and visualization tool, not a validated engineering design tool.

Core requirements:
1. The app must support uploading two Excel files:
   - Flow workbook: "Springfield mean daily flow 20-24.xlsx"
   - Water quality workbook: "Springfield raw WQ data 20-25.xlsx"

2. The app must have a navigation structure with multiple pages:
   - Home / Overview
   - Flow Analysis
   - Water Quality Analysis
   - Scenario Analysis
   - Data Upload

3. The app should automatically parse the uploaded Excel files and extract useful data.
   - For the flow workbook:
     - Read monthly overflow days
     - Read average monthly flow
     - Read monthly overflow volume
     - Read daily flow data from the detailed sheet
     - Use 25 L/s as the overflow threshold
   - For the water quality workbook:
     - Read sample date, determinand name, result, and units
     - Map determinands into four main parameters:
       - SS
       - BOD
       - Ammonia
       - COD

4. Home / Overview page:
   - Show a short project description
   - Show KPI cards such as:
     - Total flow records
     - Days above 25 L/s
     - Total overflow volume
     - Number of water quality samples
     - Number of years covered
   - Show a concise note that this is an exploratory academic dashboard

5. Flow Analysis page:
   - Monthly average flow chart
   - Monthly overflow days chart
   - Monthly overflow volume chart
   - Year-by-year comparison chart for overflow days and overflow volume
   - Seasonal summary chart
   - Threshold exceedance analysis:
     - Number of days above 25 L/s
     - Percentage above threshold
     - Monthly exceedance frequency
   - Include filters for year range
   - Include tooltips and clean chart labels

6. Water Quality Analysis page:
   - Time-series chart for each parameter
   - Distribution chart (histogram or boxplot) for each parameter
   - Yearly summary chart for mean / median / max
   - Correlation heatmap between SS, BOD, Ammonia, and COD where sampling timestamps align
   - Permit reference comparison using these limits:
     - BOD: 100 mg/L
     - COD: 340 mg/L
     - Suspended solids: 65 mg/L
     - Ammonia: 20 mg/L
   - Show how many samples exceed each permit value
   - Show exceedance percentages
   - Clearly display a disclaimer:
     "These are raw influent / pre-primary data and are shown for reference comparison only. They do not represent final discharge compliance."
   - Include parameter selector and year filter

7. Scenario Analysis page:
   - Future flow scenario tool:
     - Let user apply a percentage increase to all flow values (e.g. +5%, +10%, +20%)
     - Recalculate days above 25 L/s
     - Recalculate estimated overflow volume
     - Visualize baseline vs scenario
   - Treatment efficiency scenario tool:
     - Let user manually input removal efficiencies for SS, BOD, COD, and Ammonia
     - Estimate post-treatment concentrations with:
       estimated_output = input_value * (1 - removal_efficiency)
     - Compare estimated output to permit limits
     - Show percentage of samples that would theoretically meet limits
     - Clearly show this disclaimer:
       "This is a simplified exploratory scenario tool, not a validated engineering compliance model."

8. Data Upload page:
   - Drag-and-drop upload area
   - Show upload status
   - Show detected file type
   - Allow replacing uploaded files
   - Handle parsing errors gracefully

9. UI/UX requirements:
   - Clean modern dashboard style
   - Responsive layout
   - Sidebar or top navigation
   - Card-based sections
   - Consistent colors and spacing
   - Easy export or download of summary data / charts if possible

10. Technical preferences:
   - Use React + TypeScript
   - Use Tailwind CSS
   - Use a good chart library such as Recharts or Plotly
   - Use xlsx / SheetJS for Excel parsing
   - Keep code modular and readable
   - Separate logic into components, utilities, and parsing helpers

11. Important notes:
   - This app is for academic presentation and exploratory analysis
   - Avoid pretending the scenario results are fully validated engineering outputs
   - Add clear labels and disclaimers where needed
   - Focus on clarity, strong visualization, and presentation value

Please generate:
1. A complete project structure
2. React components for each page
3. Excel parsing utilities
4. Example mock data fallback if upload is missing
5. A polished UI suitable for a presentation demo
